Temporary Docker Container  

docker run -it --network=host --rm debian:stable-slim bash

This command runs the latest, stable, slim image of debian connected to the host's network in a interactive bash terminal. So that one can test or work with a temporary environment.


Flag Breakdown

-i Specifies that the container should be run in interactive mode, so one can work inside of it.

-t Tells docker to connect the container to our terminal via TTY.

--network=host Connects the container to the hosts network with the docker proxy.

--rm Removes the container after it is done running, effectively making it temporary.

debian:stable-slim This is the image we want to start with, in our case it is a slim version of debian.

bash Is the command or program we want to start from, in this case, the bash shell.
